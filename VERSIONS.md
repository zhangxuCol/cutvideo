# V6 视频重构算法 - 版本说明

| `v6_original.py` | V6原始版本 | 基础算法，保留原始逻辑 |
| `v6_precision.py` | V6高精度版 | 98%阈值，5点验证，追求质量 |
| `v6_fast.py` | V6极速版 | 3分钟/视频，预计算+并行 |

---

## 🔧 版本对比

### V6 Original（原始版）
```python
# 特点
- 分段时长：30s
- 匹配阈值：85%
- 验证点：1个（起始位置）
- 处理时间：10-15分钟/视频
- 音画同步：❌ 存在问题
```

**问题**：
1. 音画不同步（多源拼接时）
2. 匹配精度不够（165秒处失败）
3. 处理速度慢

---

### V6 Precision（高精度版）
```python
# 优化点
- 分段时长：10s（更细粒度）
- 匹配阈值：98%（更严格）
- 验证点：5个（起始、25%、50%、75%、结束）
- 处理时间：~15分钟/视频
- 音画同步：✅ 双重验证
```

**改进**：
1. 提高匹配阈值到98%
2. 增加多点画面验证
3. 强制时长对齐
4. **目标**：100%通过一致性检查

**代价**：处理时间更长

---

### V6 Fast（极速版）⭐
```python
# 优化点
- 分段时长：10s
- 匹配阈值：95%（平衡）
- 验证点：3个（起始、50%、结束）
- 处理时间：~3分钟/视频 ⭐
- 音画同步：✅ 预计算优化
```

**核心技术**：
1. **预计算音频指纹**
   - 首次提取后缓存到磁盘
   - 后续直接加载，节省80%时间
   
2. **多线程并行**
   - 4线程同时处理多个段
   - CPU利用率提升4倍
   
3. **智能预筛选**
   - 音频快速匹配定位候选
   - 仅对高分候选进行画面验证
   
4. **降低采样率**
   - 音频从8000Hz降到4000Hz
   - 帧验证从5点减到3点

## 🎯 使用建议

### 选择指南

```
需要处理10个视频？
├── 质量要求高（如115196有问题）
│   └── 使用 v6_precision.py
│       └── 10 × 15min = 150分钟
│
└── 效率要求高（批量处理）
    └── 使用 v6_fast.py ⭐
        └── 10 × 3min = 30分钟
```

### 推荐工作流

1. **首次处理**：使用 v6_fast.py 快速处理
2. **质量检查**：运行一致性检查
3. **问题修复**：对未通过的使用 v6_precision.py 重处理
4. **指纹缓存**：v6_fast.py 的缓存可复用

## 💡 命名由来

**为什么是V6？**
- V1-V5：实验版本，已废弃
- V6：当前稳定版基础算法
- V6.x：基于V6的优化变种

**为什么不直接叫V7？**
- 核心算法未变（仍是V6）
- 只是性能/精度优化
- 类似：Python 3.8 vs 3.9
