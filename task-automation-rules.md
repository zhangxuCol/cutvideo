# 任务自动化规则文档

## 任务执行触发条件

### 任务1: 运行V6 Fast测试
**触发条件**: 代码优化完成
**执行动作**: 
1. 运行 `v6_fast.py` 处理待处理视频
2. 记录处理时间和输出文件

**完成标准**: 
- 视频成功生成
- 处理时间 < 3分钟/视频

---

### 任务2: 音视频一致性验证（100%通过）
**触发条件**: 视频裁剪/处理完成
**执行动作循环**:
1. 每隔5秒提取视频帧和音频片段
2. 验证音视频同步（画面内容与音频内容匹配）
3. 输出对比结果（通过率百分比）
4. 检查是否100%通过

**分支逻辑**:
- ✅ **100%通过** → 任务完成，标记为成功
- ❌ **未通过** → 自动进入优化循环：
  - 分析失败时间点
  - 优化代码（调整同步参数/修复算法）
  - 重新运行处理
  - 再次验证
  - 重复直至100%通过

**关键规则**:
- 验证画面内容时AI亲自查看，无需用户确认
- 自动迭代优化，直到完全通过
- 每次迭代记录优化点和通过率变化

---

### 任务3: 高精度版验证（可选）
**触发条件**: V6 Fast版本多次优化后仍无法通过
**执行动作**: 
1. 切换到 `v6_precision.py`（98%阈值）
2. 运行高精度处理
3. 重新进行一致性验证

---

## 任务执行摘要记录格式

每个任务在摘要中应包含：
- 任务名称和规则简述
- 触发条件
- 当前执行状态
- 自动化执行次数/迭代次数
- 最终结果（通过/失败）
- 遇到的问题和解决方案

示例：
```
## 待自动执行任务

### 任务: 115196视频一致性验证
**规则**: 裁剪完成后每5秒验证音视频同步，未通过则自动优化代码并重试直至100%
**触发**: 视频115196_V3_FAST.mp4生成完成
**状态**: 🔄 自动执行中（第3轮迭代）
**当前通过率**: 97.5% (39/40检查点)
**问题**: 165秒处场景切换导致音画不同步
**优化**: 调整场景检测阈值，增加过渡帧处理
**下次执行**: 立即
```

## 技术实现建议

1. **任务状态追踪**: 使用 bitable 或本地 JSON 记录每个任务的执行状态
2. **自动化触发**: 通过 cron 或文件系统监听(watchdog)检测视频生成
3. **验证工具**: 使用现有的 `av_consistency_checker.py`
4. **自动优化**: 需要开发代码自动调参/修复逻辑
5. **日志记录**: 详细记录每次迭代的修改和结果
